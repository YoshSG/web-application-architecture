Arquitectura de Aplicaciones Web


  • Capa de Presentación

    • El frontend de una aplicación web moderna se basa en frameworks SPA como React y Angular, complementados con SSR y tecnologías como GraphQL o WebSockets para la actualización en tiempo real.


  • Capa de Lógica de Negocio

    • La lógica de negocio se diseña en base a arquitecturas de microservicios con orquestación mediante Kubernetes, favoreciendo la independencia de despliegue y la escalabilidad horizontal.


  • Capa de Almacenamiento

    • Para aplicaciones distribuidas, las bases de datos NoSQL como MongoDB o bases SQL distribuidas como Google Spanner son ideales para gestionar la disponibilidad global y la consistencia eventual.

Herramientas y Metodologías de Planificación


Para asegurar un proceso de planificación eficaz, existen múltiples herramientas y metodologías que pueden ayudar a gestionar el proyecto y coordinar al equipo de desarrollo. Algunas de las herramientas más populares incluyen:


  • Trello: Una herramienta de gestión de proyectos que permite organizar tareas en tableros visuales.

  • Asana: Ideal para coordinar equipos y realizar seguimiento de proyectos a lo largo del tiempo.

  • Jira: Herramienta orientada al desarrollo ágil, útil para gestionar el ciclo de vida de un proyecto de software.

  • Lucidchart: Utilizada para diseñar diagramas de flujo y arquitectura.

En cuanto a metodologías de planificación, algunas de las más comunes son:


  • Agile: Un enfoque iterativo que permite la entrega constante de pequeñas mejoras y funcionalidades.

  • Scrum: Una metodología ágil que se organiza en sprints, ciclos cortos de trabajo enfocados en lograr entregas parciales y funcionales.

  • Kanban: Visualiza el flujo de trabajo y permite realizar ajustes para evitar cuellos de botella.

  • Waterfall: Un enfoque más tradicional, donde el proyecto se desarrolla en etapas secuenciales.

Hardware a utilizar


Primera opción: Servidor físico propio


La instalación de un servidor físico en las instalaciones del TESE garantiza un control total sobre la infraestructura, lo que es ideal si se desea una administración directa de los recursos de hardware y mayor personalización.

    Características:


    • Procesador: Intel Xeon E5-2670. Este procesador está diseñado para entornos de servidores, ofreciendo una excelente capacidad para gestionar múltiples tareas simultáneas (multithreading) y gran rendimiento.

    • Memoria RAM: 64 GB de RAM. Cantidad suficiente para soportar varios usuarios concurrentes accediendo a la aplicación sin comprometer el rendimiento.

    • Almacenamiento: 1 TB en discos SSD. Los discos SSD permiten un acceso rápido a los datos, lo que es crucial para una aplicación web que gestiona grandes volúmenes de información académica.

    • Red: Conectividad Gigabit Ethernet. Es necesario tener una red de alta velocidad para garantizar que el acceso a la aplicación web sea rápido y sin interrupciones.

    Ventajas:


    • Control total: El equipo de TI del TESE puede tener un control directo sobre la configuración, seguridad y mantenimiento del servidor.

    • Personalización: Se puede ajustar el hardware de acuerdo con las necesidades específicas del proyecto sin depender de un proveedor externo.


    Desventajas:


    • Costo inicial elevado: Comprar servidores físicos implica una inversión inicial considerable, además del espacio físico necesario para albergarlos.

    • Mantenimiento continuo: Requiere un equipo especializado para el monitoreo y actualización continua del hardware.

Segunda opción: Servidores en la nube (AWS - Amazon Web Services)


AWS ofrece una infraestructura flexible y escalable, ideal para proyectos que necesitan un crecimiento rápido o que no tienen el capital para invertir en servidores físicos.

Características:


  • Instancia EC2 (Elastic Compute Cloud): Proporciona capacidad de procesamiento bajo demanda, con 16 vCPUs y 32 GB de RAM, suficiente para soportar la carga inicial de la aplicación.

  • Almacenamiento: Uso de Elastic Block Store (EBS) con 500 GB de almacenamiento. EBS ofrece alta disponibilidad y redundancia de datos para garantizar que no haya pérdida de información.

  • Escalabilidad automática: AWS permite aumentar o disminuir los recursos automáticamente en función de la demanda de usuarios, lo que es esencial para aplicaciones que podrían tener picos de uso.

Ventajas:


  • Pago por uso: En lugar de realizar una inversión inicial, solo se paga por los recursos utilizados. Esto permite flexibilidad en costos, especialmente para instituciones que no cuentan con un gran presupuesto inicial.

  • Escalabilidad: La capacidad de aumentar los recursos de forma dinámica es ideal si se espera un crecimiento en la cantidad de usuarios o la demanda de la aplicación.


Desventajas:


  • Dependencia de un proveedor externo: Si bien AWS es fiable, siempre se dependerá de un tercero para el mantenimiento de los servidores y la infraestructura.

  • Costos variables: Los costos pueden aumentar si la demanda de recursos es alta, lo que debe ser monitoreado cuidadosamente para evitar sobrecostos.
